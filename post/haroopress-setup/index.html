haroopress setup

Haroopress 설치기

하루프레스는 마크다운을 기반으로한 블로그 포스팅 툴로서 워드프래스와 같은 CMS(Contents Management System) Tool이지만 개발자들이 익숙한 콘솔기반의 프래싱 툴이라고 할 수 있다. 즉, 콘솔에 익숙하지 않은 분들은 사용하기 오히려 어렵거나 이해하기 어려울 수 있다. 즉 네이버에서 글쓰듯 위지위그 환경이 제공되지 못하고 기본적으로 Git 에 대한 이해가 필수 이다 보니 더더욱 그러한 듯 하다.

우선 설치 전에 필요한 환경과 필수요소를 정리한다.

하루프레스의 제작자 홈페이지에 잘 셜명되어 있으나, 조금은 어려운 듯 해서 조금더 쉽게 작성해 보기로 한다.

본 문서의 용도는 다른 사람을 위하기 보다는 필자 본인의 경험을 남겨 놓기 위한 용도에 더욱 가깝다는 점을 우선 밝힌다.

하루프레스 동작방법

하루프레스는 기본적으로 로컬에서 동적페이지를 생성하고 preview 하는 방식으로 운용하고 이를 외부에 업로드(?) 하는 형태로 운용되는 방식입니다.

이때 외부에 업로드하는 방법으로 현재까지는 Github만 지원하고 있습니다.

필요환경 및 필수요소

필요환경

  • 운영체제

하루프레스 홈페이지에 아직까지(2014.01.31) Mac환경만 지원합니다.

  • Node.js

하루프레스는 자바스크립 기반의 정적페이지 생성을 통한 블로깅 툴이며 Node.js를 반드시 필요로 합니다. 홈페이지에서는 0.6 이상의 최신버젼을 요구 합니다.

다만, Node.js의 소스설치보다는 바이너리 설치를 관리해 주는 노드 버젼 메니저도 함께 설치해야 하는 듯 합니다. 각각의 설치파일 다운로드 페이지 입니다.

http://nodejs.org

https://github.com/creationix/nvm

  • Git

하루프레스는 다른 Git 서브 모듈로 운용된다고 합니다. 설치 합시다.

http://git-scm.com/

Github 세팅

$ make gh-pages

Github 주소를 입력하니다. HTTS방식일 경우 https://github.com/ypkim/ypkim.github.com.git 형태로 작성합니다.

글 작성하는 방법

$ make gen       // 정적 페이지 생성
$ make preview   // 미리보기
$ make new-post  // 새로운 글 작성
$ make new-page  // 새로운 페이지 작성
$ make new-slide // 새로운 웹 슬라이드 작성

$ make new-post 를 통해 글을 작성하게 되면 초기에는 파일 상태가 "draft"상태 입니다. 이를 "publish" 로 바꿔줘야 정상적으로 발행 됩니다. 바꾸는 위치는 각 마크다운 파일을 열어서 다음의 내용을 확인 합니다.

"status": "publish",

글 올리는 방법

$ make deploy

위방법으로 Github에 publish 됩니다.
